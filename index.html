Apex Track Live

High-frequency telemetry analysis for the next generation of professional racing.

The Architecture

We built a proprietary data pipe using binary serialization instead of JSON. This allowed the app to ingest 1,000+ data points per second (GPS, Throttle, Brake, Steering Angle) with less than 80ms total latency from car to screen.

Native Performance

Leveraging Apple's Metal framework, the app renders complex track geometry and G-force maps entirely on the GPU. This ensures the main UI thread remains fluid even during high-intensity data bursts.

Project Stack
  • Platform Swift / SwiftUI
  • Real-time WebSockets
  • Rendering Metal API

App Privacy

Apex Track Live: Privacy Policy

1. Telemetry Data Handling

Apex Track Live processes race data. By default, this data is processed locally on your device. We do not transmit live telemetry to our servers. All data gets transfered via a direct connection to the data API.

2. Internet Data

The App requires high-speed internet to function. This data is used only for data transfer. We do not use your internet data to track your location or any other similar type of data.

3. Third-Party Integrations

If you connect third-party hardware (e.g., Bluetooth OBD-II dongles), data is retrieved directly by the App. Refer to your hardware manufacturer's privacy policy for their specific data handling practices.

4. Data Deletion

Users can delete all recorded sessions and telemetry data directly within the App settings. Once deleted, this data is irrecoverable as we do not maintain persistent backups of user telemetry.
